# Curated v1.4.8

Curated v1.4.8 adds the PIN App Lock security flow, shows connected clients in Settings, improves desktop display density and Curated Frames browsing, and restores authenticated media playback by sending credentials on direct video and HLS requests.

Build stamp: `20260516.174515`

## Highlights

- Added PIN App Lock with backend security storage, lock/unlock APIs, idle locking, a dedicated lock screen, and Settings controls for enabling, changing, and disabling the PIN.
- Added the Connected Clients Settings panel with tracked browser and desktop clients, including Electron desktop OS markers sent by the desktop shell.
- Restored authenticated playback under the PIN lock/Web API flow by using credentialed direct video requests, HLS XHR credentials, and credentialed HLS manifest probing.
- Improved desktop display density for larger Windows desktop surfaces, including sidebar, grid, card, and masonry sizing adjustments.
- Refined Curated Frames browsing with a carousel-backed detail experience, denser frame cards, dialog navigation helpers, preview image zoom/pan improvements, and notification behavior fixes.
- Removed the Electron desktop menu bar for the packaged shell and fixed the desktop marker header typing issue that blocked the first v1.4.8 Electron main build attempt.
- Updated API, architecture, feature, and planning documentation for PIN lock, connected clients, desktop clients, and display density behavior.
