# Curated v1.4.1

This release is a larger feature and stability package. It adds browser-based movie import workflows, daily watch-time statistics, notification-center improvements, and a broad frontend service-layer hardening pass.

Build stamp: `20260502.093720`

## Highlights

- Added top-bar movie import into the configured default library path, including file/folder selection, copy-only semantics, progress reporting, partial-failure handling, and scan-task handoff.
- Added resumable large-file movie import upload sessions under `/api/import/movies/uploads`, with chunk upload, status, commit, abort, staging cleanup, and retryable frontend upload handling.
- Added Settings -> Library & storage support for choosing the default import folder, persisted through `defaultImportLibraryPathId`.
- Added daily playback watch-time tracking. Web API mode stores bounded per-day watch deltas in SQLite, while mock mode uses `localStorage`; Settings -> Overview displays the 91-day summary.
- Added a notification center and improved task/toast handling for library watch scans, movie imports, success states, partial failures, and failed background work.
- Hardened API and frontend service behavior with response validation, client request timeouts, direct loopback Web API routing for large local uploads, safer upload diagnostics, and expanded Web/Mock adapter coverage.
- Continued frontend cleanup and performance work across the shell, Settings sections, library service routing, i18n coverage, player state, curated-frame components, and error handling.
- Fixed the repo-local packaging skill preview so it reads the active package ledger at `docs/ops/package-build-history.csv`.
