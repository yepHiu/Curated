# Curated v1.3.4

This release focuses on backend robustness, production packaging reliability, desktop Web regression coverage, and a small player control polish pass.

## Highlights

- Hardened backend request handling and storage behavior, including request body limits, safer reveal-path errors, atomic config write retry simplification, sentinel-based error classification, cancellation-aware scanner walking, and safer scrape shutdown behavior.
- Improved homepage recommendation candidate selection performance.
- Added a reusable Curated dev-start skill and production packaging workflow guidance.
- Added desktop Web display scaling smoke coverage for Chrome and Firefox while pausing the broader multi-device rollout until the desktop Web scope is settled.
- Added an explicit rule that `pnpm test:display` and other display-scaling smoke tests require user consent before running.
- Simplified the player native-player control to an icon-only button with an accessible label.
- Exposed the Vite dev server on the local network for easier LAN device testing during development.

## Notes

- The Windows package includes FFmpeg runtime files under `third_party/ffmpeg/`; keep the included upstream license notice when redistributing the package.
- If you use the portable package, extract it to a clean folder before running `curated.exe`.
- The bundled release backend runs as the production identity `curated` and uses the release HTTP port/config defaults.
