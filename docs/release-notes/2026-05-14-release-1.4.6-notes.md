# Curated v1.4.6

Curated v1.4.6 adds the first in-repository Electron desktop shell MVP, improves desktop/web runtime lifecycle behavior, and includes fixes for library count handling and homepage recommendation snapshots.

Build stamp: `20260513.180832`

## Highlights

- Added an Electron desktop shell MVP with Curated icon integration, secure BrowserWindow defaults, single-instance restore, and a tray menu for reopening Curated, opening the Web UI in a browser, opening Settings, or quitting.
- Added close-to-tray behavior: closing the desktop window hides it while Electron-owned backend/frontend processes keep running; choosing Quit from the tray performs the real shutdown.
- Added Electron-managed backend startup/attach support. Electron waits for `/api/health`, reports a ready event after the Go HTTP listener is actually bound, and only stops the backend if Electron spawned it.
- Added Electron development frontend management. `pnpm dev:electron` starts or reuses Vite at `http://127.0.0.1:5173`, points it at the Electron-managed backend, and keeps packaged runtime on backend-hosted static UI at `http://127.0.0.1:8081`.
- Added the narrow preload bridge `window.javLibrary.pickDirectory()` so existing folder-picking flows can use Electron's native directory dialog without moving business APIs to IPC.
- Fixed homepage daily recommendations so backend snapshots can preserve the full recommendation list size instead of truncating to the local fallback model length.
- Included the previous library count fix for large scan and list counts.

## GitHub Release Body

Curated v1.4.6 adds the first in-repository Electron desktop shell MVP, improves desktop/web runtime lifecycle behavior, and includes fixes for library count handling and homepage recommendation snapshots.

### Highlights

- Electron desktop shell MVP is now in the repository: it starts or reuses the Go HTTP backend, opens the existing Web UI in a secure BrowserWindow, uses the Curated app icon, restores on second launch, and keeps business APIs on HTTP rather than broad IPC.
- Closing the Electron window now hides Curated to the tray. The backend and Web entry stay available until the user chooses Quit from the tray.
- The tray menu provides Open Curated, Open Web App in Browser, Open Settings, and Quit Curated.
- `pnpm dev:electron` now starts or reuses both the Go backend on `:8080` and Vite on `:5173`; packaged runtime still uses the backend-hosted static UI on `:8081`.
- Existing directory selection flows can use Electron's native folder picker through `window.javLibrary.pickDirectory()`.
- Homepage daily recommendation snapshots now preserve the full backend-provided recommendation size.
- Large library scan/list count handling from the previous post-1.4.5 fix is included.
