# Curated v1.3.3 Release Notes Draft

## Summary

Curated v1.3.3 focuses on homepage recommendation freshness and LAN packaged-app access. This release adds persisted per-movie recommendation state for homepage hero and today's picks, switches recommendation assembly to weighted sampling with hard cooldowns and recommendation-count decay, keeps same-day snapshots stable across browsers and devices, and makes the bundled frontend use same-origin `/api` by default so LAN clients opening port `8081` can reach the backend correctly.

## Highlights

- Added `homepage_recommendation_states` SQLite persistence for per-movie recommendation memory, including last recommendation time, recommendation count, hard-cooldown expiry, and update timestamp.
- Changed homepage hero and today's picks generation from deterministic top-ranked selection to weighted sampling without replacement.
- Added hard cooldown and 14-day recovery behavior so recently recommended movies are skipped or down-weighted while long-unseen titles become more likely to surface.
- Added logarithmic recommendation-count decay so frequently surfaced titles lose priority over time and long-tail titles get more opportunities.
- Preserved same-day daily snapshots as the display cache so refreshes and multiple devices still see the same daily homepage slate.
- Regenerated stale same-day homepage snapshots when `generationVersion` changes, preventing old recommendation algorithms from lingering for the rest of the day.
- Kept recent daily snapshot exclusion and actor/studio diversity penalties in the homepage slate builder.
- Changed the packaged frontend API default to same-origin `/api`, allowing LAN devices to open `http://<host-ip>:8081` and call the backend on the same host instead of trying their own localhost.
- Ignored generated `backend/frontend-dist/` assets in source control; release packaging still builds and embeds production frontend assets into the release directory.

## Curated v1.3.3
