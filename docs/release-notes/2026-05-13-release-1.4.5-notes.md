# Curated v1.4.5

Curated v1.4.5 extends the in-app update flow with verified installer download, explicit installation, and an opt-in General settings toggle that can prepare new installers in the background.

Build stamp: `20260512.174356`

## Highlights

- Added in-app installer download from Settings -> About, with SHA256 verification before installation is offered.
- Added explicit installer launch after a verified download is ready; Curated still requires user confirmation before starting the installer.
- Added Settings -> General -> App updates -> Automatically download available updates, persisted as `autoDownloadUpdates` in `config/library-config.cfg`.
- When `autoDownloadUpdates` is enabled, the startup background update check can automatically download and SHA256-verify a newer installer. It does not auto-install.
- Updated HTTP, stdio, frontend service, Mock/Web adapters, and project documentation for the new settings contract.

## GitHub Release Body

Curated v1.4.5 extends the in-app update flow with verified installer download, explicit installation, and an opt-in General settings toggle that can prepare new installers in the background.

### Highlights

- Settings -> About can now download the latest Windows installer inside Curated, verify it with SHA256, and then show an explicit "Install now" action after verification succeeds.
- Settings -> General now includes "Automatically download available updates". The setting is persisted as `autoDownloadUpdates` in `config/library-config.cfg`.
- When that toggle is enabled, the startup background update check can automatically download and SHA256-verify a newer installer. Installation still requires user confirmation.
- The settings API, stdio settings DTO, frontend service contract, Mock/Web adapters, tests, README, API docs, CLAUDE notes, and project facts now describe the new update setting.
- The previous update behavior remains available: users can still manually check for updates, download/install from About, or open the GitHub Release page.
