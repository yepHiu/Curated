# Curated v1.4.9

Curated v1.4.9 removes the prominent Curated Frames duplicate-review summary alert from the library page while keeping the underlying near-duplicate detection data available to the frame UI. It also hardens release packaging for constrained Windows agent environments by allowing the Go release build cache and temp directories to be supplied explicitly.

Build stamp: `20260606.170226`

## Highlights

- Removed the top-level Curated Frames duplicate-review summary alert from the frame library layout to reduce visual noise during normal browsing.
- Kept near-duplicate detection indexing in place so frame-level duplicate affordances and related logic can continue to operate.
- Added a focused regression test that verifies the duplicate-review summary alert is not rendered by `CuratedFramesLibrary.vue`.
- Hardened the release backend build step with `CURATED_RELEASE_GOCACHE` and `CURATED_RELEASE_GOTMPDIR`, allowing packaging agents to use stable writable Go build directories when system temporary directories are restricted.
- Preserved package build history audit rows for the failed temp-directory attempts and the final successful `1.4.9` publish run.
