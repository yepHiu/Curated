## 项目概述

本项目为 Curated（jav-shadcn）桌面媒体库应用建立完整的持续集成与持续部署（CI/CD）流程。考虑到项目采用 Go 后端 + Vue 3 前端的技术栈，且最终目标是构建 Electron 桌面应用，CI/CD 流程需要支持多平台构建（Windows、macOS、Linux）和多种发布渠道（Web、桌面应用）。

### 当前状况

| 维度 | 现状 | 目标 |
|------|------|------|
| 测试 | 手动运行，覆盖率未知 | 每次提交自动运行，覆盖率可见 |
| 构建 | 本地手动构建 | 自动化多平台构建 |
| 发布 | 无 | GitHub Releases 自动发布 |
| 代码质量 | 依赖人工审查 | 自动化 lint + 审查 |
| 假数据测试 | 无 | 集成假库自动生成 |

## 目标与收益

### 核心目标

1. **每次提交质量保证**
   - 自动运行单元测试和集成测试
   - 代码质量检查（lint、类型检查）
   - 测试覆盖率报告

2. **自动化构建**
   - 前端自动构建和打包
   - 后端多平台交叉编译
   - Electron 应用多平台打包

3. **持续交付**
   - 主分支自动发布到 GitHub Releases
   - 开发分支自动生成预览版本
   - 版本号自动管理

4. **假库集成测试**
   - CI 中自动生成测试数据库
   - 大规模数据性能测试
   - 端到端场景验证

### 预期收益

- **开发效率提升**: 减少 60% 的手工测试时间
- **缺陷提前发现**: 在合并前捕获 80% 的回归问题
- **发布周期缩短**: 从手动数小时缩短到自动 15 分钟
- **协作信心**: 团队成员可以安全地合并代码

## 技术选型

### CI/CD 平台

| 平台 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **GitHub Actions** | 与 GitHub 原生集成，免费额度充足，生态丰富 | 大型构建可能较慢 | ⭐⭐⭐⭐⭐ |
| GitLab CI | 功能强大，自建 Runner 灵活 | 需要维护基础设施 | ⭐⭐⭐ |
| CircleCI | 速度快，并行能力强 | 免费额度有限 | ⭐⭐⭐ |

**推荐**: GitHub Actions（原生支持，无需额外配置）

### 关键工具

| 用途 | 工具 | 说明 |
|------|------|------|
| 测试框架 | Go test + Vitest | 原生支持，无需额外配置 |
| 覆盖率 | Codecov / Coveralls | 可视化覆盖率报告 |
| 代码质量 | golangci-lint + ESLint | 统一代码风格 |
| 构建 | GoReleaser | 强大的 Go 多平台构建工具 |
| Electron 打包 | electron-builder | 跨平台桌面应用打包 |
| 假数据 | 内部 mock 生成器 | 参考《开发测试假库设计方案》 |
| 通知 | GitHub Issues / Slack | 构建失败通知 |

### 工作流详细设计

#### 1. PR 检查工作流 (ci-pr.yml)

**触发条件**: PR 创建、更新、同步

**任务矩阵**:

```yaml
jobs:
  # ============ 前端检查 ============
  frontend-check:
    runs-on: ubuntu-latest
    steps:
      - checkout
      - setup-node
      - pnpm install
      - lint: eslint + prettier
      - typecheck: vue-tsc
      - test: vitest run
      - coverage: 上传报告

  # ============ 后端检查 ============
  backend-check:
    runs-on: ubuntu-latest
    steps:
      - checkout
      - setup-go
      - lint: golangci-lint
      - test: go test -race -cover
      - coverage: 上传报告

  # ============ 假库集成测试 ============
  mock-integration-test:
    runs-on: ubuntu-latest
    needs: [backend-check]
    steps:
      - checkout
      - setup-go + setup-node
      - 生成假库数据 (1000 部影片)
      - 启动后端 + 前端
      - 运行 API 集成测试
      - 运行 E2E 测试 (Playwright)

  # ============ 构建验证 ============
  build-verify:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
    steps:
      - checkout
      - 构建前端
      - 构建后端（交叉编译）
      - 验证可执行文件
```

#### 2. 主分支部署工作流 (cd-release.yml)

**触发条件**: 代码合并到 master 分支、手动触发

**发布流程**:

```yaml
jobs:
  # 版本号计算
  version:
    steps:
      - 读取当前版本 (git describe)
      - 计算新版本号 (语义化版本)
      - 创建 Git tag

  # 构建 Web 版本
  build-web:
    steps:
      - 构建前端静态文件
      - 打包为可部署格式
      - 上传 artifact

  # 构建桌面应用
  build-electron:
    strategy:
      matrix:
        include:
          - os: windows-latest
            arch: x64
          - os: macos-latest
            arch: x64
          - os: macos-latest
            arch: arm64
          - os: ubuntu-latest
            arch: x64
    steps:
      - 构建前端
      - 使用 electron-builder 打包
      - 签名 (macOS/Windows)
      - 上传 artifact

  # 创建 Release
  release:
    needs: [build-web, build-electron]
    steps:
      - 创建 GitHub Release
      - 上传所有构建产物
      - 生成 changelog
      - 发布到 GitHub Packages (可选)
```

#### 3. 定时任务工作流 (ci-scheduled.yml)

**触发条件**: 每周日凌晨 2 点

```yaml
jobs:
  # 大规模性能测试
  performance-test:
    steps:
      - 生成大规模假库 (10000 部影片)
      - 运行性能基准测试
      - 生成性能报告
      - 性能回归检测

  # 依赖更新检查
  dependency-check:
    steps:
      - 检查 Go 依赖更新
      - 检查 Node 依赖更新
      - 创建 PR 更新依赖

  # 安全扫描
  security-scan:
    steps:
      - Go 依赖漏洞扫描 (govulncheck)
      - Node 依赖漏洞扫描 (npm audit)
      - 代码安全扫描 (CodeQL)
```

## 自动化测试策略

### 测试金字塔

```
                    ▲
                   / \
                  /   \
                 / E2E \          5%  (关键路径)
                /_______\
               /         \
              / Integration \     15% (API、组件)
             /_______________\
            /                 \
           /     Unit Test      \  80% (函数、逻辑)
          /_____________________\
```

### 测试分层

#### 1. 单元测试（快速，每次提交）

**前端 (Vitest)**:
```typescript
// 组件测试
import { mount } from '@vue/test-utils'
import MovieCard from '@/components/jav-library/MovieCard.vue'

describe('MovieCard', () => {
  it('renders movie title', () => {
    const wrapper = mount(MovieCard, {
      props: { movie: mockMovie }
    })
    expect(wrapper.text()).toContain(mockMovie.title)
  })
})
```

**后端 (Go test)**:
```go
// 存储层测试
func TestMovieRepository_Create(t *testing.T) {
    db := testutil.NewTestDB(t)
    repo := storage.NewMovieRepository(db)

    movie := &domain.Movie{Title: "Test"}
    err := repo.Create(context.Background(), movie)
    require.NoError(t, err)
    assert.NotZero(t, movie.ID)
}
```

#### 2. 集成测试（假库，每次 PR）

**API 集成测试**:
```go
func TestMovieAPI_CRUD(t *testing.T) {
    // 使用假库
    mockDB := mock.GenerateTestDB(t, mock.ScenarioSmall)
    server := testutil.NewTestServer(mockDB)

    // 测试完整 CRUD 流程
    resp := server.Post("/api/library/movies", createReq)
    assert.Equal(t, 201, resp.StatusCode)

    movieID := extractID(resp)
    resp = server.Get("/api/library/movies/" + movieID)
    assert.Equal(t, 200, resp.StatusCode)
}
```

#### 3. E2E 测试（关键路径，每日构建）

**Playwright 测试**:
```typescript
// e2e/library.spec.ts
import { test, expect } from '@playwright/test'

test('用户可以浏览影片库', async ({ page }) => {
  await page.goto('/library')

  // 等待影片加载
  await expect(page.locator('[data-testid="movie-grid"]')).toBeVisible()

  // 搜索功能
  await page.fill('[data-testid="search-input"]', 'ABC')
  await expect(page.locator('[data-testid="movie-card"]')).toHaveCount(3)

  // 点击查看详情
  await page.click('[data-testid="movie-card"]:first-child')
  await expect(page.locator('[data-testid="movie-detail"]')).toBeVisible()
})
```

## 配置示例

### GitHub Actions 配置

#### `.github/workflows/ci-pr.yml`

```yaml
name: CI - Pull Request

on:
  pull_request:
    branches: [master, develop]

jobs:
  # ========== 前端检查 ==========
  frontend:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: .

    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v2
        with:
          version: 8

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm lint

      - name: Type check
        run: pnpm typecheck

      - name: Unit tests
        run: pnpm vitest run --coverage

      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          files: ./coverage/lcov.info
          flags: frontend

  # ========== 后端检查 ==========
  backend:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: ./backend

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-go@v4
        with:
          go-version: '1.22'
          cache: true

      - name: Install golangci-lint
        uses: golangci/golangci-lint-action@v3
        with:
          version: latest
          working-directory: ./backend

      - name: Run tests
        run: go test -race -coverprofile=coverage.out ./...

      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          files: ./backend/coverage.out
          flags: backend

  # ========== 假库集成测试 ==========
  integration:
    runs-on: ubuntu-latest
    needs: [frontend, backend]

    steps:
      - uses: actions/checkout@v4

      # 设置 Go 和 Node 环境...

      - name: Generate mock library
        run: |
          cd backend
          go run ./cmd/curated -seed-db medium -mock-path /tmp/mock-lib

      - name: Start backend
        run: |
          cd backend
          go run ./cmd/curated -db /tmp/mock-lib/test.db &
          sleep 5

      - name: Run API tests
        run: |
          cd backend
          go test -tags=integration ./test/integration/...

  # ========== 构建验证 ==========
  build:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]

    steps:
      - uses: actions/checkout@v4

      - name: Build frontend
        run: |
          pnpm install
          pnpm build

      - name: Build backend
        run: |
          cd backend
          go build ./cmd/curated
```

#### `.github/workflows/cd-release.yml`

```yaml
name: CD - Release

on:
  push:
    branches: [master]
  workflow_dispatch:

jobs:
  version:
    runs-on: ubuntu-latest
    outputs:
      version: ${{ steps.version.outputs.version }}
      release_url: ${{ steps.release.outputs.upload_url }}

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Calculate version
        id: version
        run: |
          # 基于 commit 数量计算版本号
          COMMITS=$(git rev-list --count HEAD)
          echo "version=1.0.${COMMITS}" >> $GITHUB_OUTPUT

      - name: Create Release
        id: release
        uses: actions/create-release@v1
        with:
          tag_name: v${{ steps.version.outputs.version }}
          release_name: Release ${{ steps.version.outputs.version }}
          draft: false
          prerelease: false

  build-electron:
    needs: version
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        include:
          - os: windows-latest
            platform: win32
            arch: x64
          - os: macos-latest
            platform: darwin
            arch: x64
          - os: macos-latest
            platform: darwin
            arch: arm64
          - os: ubuntu-latest
            platform: linux
            arch: x64

    steps:
      - uses: actions/checkout@v4

      - name: Build Electron app
        run: |
          pnpm install
          pnpm build:electron --${{ matrix.platform }} --${{ matrix.arch }}

      - name: Upload to Release
        uses: actions/upload-release-asset@v1
        with:
          upload_url: ${{ needs.version.outputs.release_url }}
          asset_path: ./dist/Curated-*
          asset_name: Curated-${{ needs.version.outputs.version }}-${{ matrix.platform }}-${{ matrix.arch }}.exe
          asset_content_type: application/octet-stream
```

### GoReleaser 配置

#### `.goreleaser.yml`

```yaml
project_name: curated-backend

before:
  hooks:
    - go mod tidy
    - pnpm install
    - pnpm build

builds:
  - id: curated
    main: ./cmd/curated
    binary: curated
    env:
      - CGO_ENABLED=1
    goos:
      - linux
      - windows
      - darwin
    goarch:
      - amd64
      - arm64
    ignore:
      - goos: windows
        goarch: arm64
    ldflags:
      - -s -w -X main.version={{.Version}} -X main.commit={{.Commit}}

archives:
  - format: tar.gz
    name_template: >-
      {{ .ProjectName }}_
      {{- title .Os }}_
      {{- if eq .Arch "amd64" }}x86_64
      {{- else if eq .Arch "386" }}i386
      {{- else }}{{ .Arch }}{{ end }}
    format_overrides:
      - goos: windows
        format: zip

checksum:
  name_template: 'checksums.txt'

snapshot:
  name_template: "{{ incpatch .Version }}-next"

changelog:
  sort: asc
  filters:
    exclude:
      - '^docs:'
      - '^test:'
      - '^chore:'
      - Merge pull request
      - Merge branch
```

## 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| CI 构建时间过长 | 中 | 中 | 并行化任务，使用缓存，优化测试 |
| 假库数据生成慢 | 中 | 中 | 预生成数据库快照，Git LFS 管理 |
| Electron 构建复杂 | 高 | 中 | 分阶段实施，先 Web 后桌面 |
| 多平台兼容性问题 | 中 | 高 | 使用 Docker 标准化环境 |
| 密钥/证书管理 | 中 | 高 | 使用 GitHub Secrets，不提交敏感信息 |
| 成本超出预期 | 低 | 中 | 监控 GitHub Actions 使用，考虑自托管 Runner |
