# Curated 导入影片 UI 方案草稿

日期：2026-05-01

## 背景

当前 Curated 的资料库路径在“设置 → 资料库 → 存储目录”中维护，顶栏右侧已有明暗模式切换。用户希望在明暗模式旁边新增“添加影片”入口，点击后通过弹窗拖拽或选择文件/文件夹，并把影片复制到默认影片存储路径中。

## 推荐 UI

顶栏右侧新增一个主按钮“添加影片”，放在明暗模式切换左侧。它是高频入口，但不进入侧栏导航，避免把“导入动作”伪装成一个页面。

弹窗采用两栏结构：

- 左侧：拖拽区域、选择影片、选择文件夹、待导入文件列表。
- 右侧：导入摘要、目标路径、目标结构预览、导入后动作开关。
- 底部：任务说明、取消、开始导入。

设置页的“影片存储路径”卡片新增“默认导入路径”选择。已有多个库路径时，用户显式选择一个默认导入目标；没有默认值时，导入弹窗提示先选择目标路径，或跳转设置页。

## 交互流程

1. 用户点击顶栏“添加影片”。
2. 弹窗打开，显示拖拽区和默认导入目标。
3. 用户拖入文件、拖入文件夹、选择影片或选择文件夹。
4. 前端列出识别到的视频文件，展示文件名、大小、推断番号、目标目录预览。
5. 用户确认目标路径和导入选项。
6. 点击“开始导入”，创建后台导入任务。
7. 任务复制文件到目标库路径，完成后触发扫描；如果开启元数据刮削，沿用现有元数据策略排队。
8. 进度通过现有任务进度 Dock 或 toast 反馈。

## 复制进度反馈

用户需要在三个层级知道复制进度：

- 弹窗内：开始导入后，右侧导入设置区域切换为进度摘要，显示总进度、当前文件、已复制大小、总大小、预计剩余时间、当前阶段。
- 全局任务 Dock：用户关闭或最小化弹窗后，底部任务进度 Dock 继续显示“导入影片进行中”，包含百分比、当前文件和阶段。点击 Dock 可重新打开导入详情。
- 完成/失败通知：复制完成后 toast 提示“已复制，正在扫描”或“导入完成”；失败时显示失败文件、原因和重试入口。

进度模型建议区分两类进度：

- 文件复制进度：`copiedBytes / totalBytes`，用于展示真实复制百分比。
- 导入流程阶段：复制文件、扫描资料库、刮削元数据。复制完成后百分比不能停在 100% 后无反馈，应切换到“正在扫描/正在刮削”的阶段状态。

如果后端暂时拿不到准确字节进度，MVP 也应至少显示当前文件、已完成文件数、总文件数和阶段状态；但推荐后端任务模型直接暴露字节级进度，因为大视频复制可能持续数分钟。

## 失败处理与通知机制

现有代码已经有可复用的通知和任务机制：

- `src/composables/use-app-toast.ts` 提供 `pushAppToast()`，支持 `success`、`warning`、`destructive` 等通知类型。
- `src/composables/use-scan-task-tracker.ts` 可轮询 `GET /api/tasks/{taskId}`，在任务进入 `completed`、`failed`、`partial_failed`、`cancelled` 后显示 toast。
- `src/components/jav-library/ScanProgressDock.vue` 已有底部任务进度 Dock，可展示任务进度和失败消息。
- 后端 `backend/internal/tasks/manager.go` 已有 `Create`、`Start`、`ProgressWithMetadata`、`Complete`、`Fail`，`TaskDTO` 支持 `status`、`progress`、`message`、`errorCode`、`errorCategory`、`errorMessage`、`metadata`。

导入复制需要在现有机制上新增更细的失败模型：

- 单文件失败不应立刻让整批导入完全失败，除非目标路径不可写、空间不足、任务被取消等全局错误。
- 对多文件导入，部分文件复制失败时，任务状态应为 `partial_failed`，成功复制的文件继续进入扫描。
- `metadata.errorItems` 应记录失败项列表，例如源文件名、目标路径、错误码、错误消息、是否可重试。
- 弹窗内显示“失败项”区域，允许用户查看失败原因，并对可重试项重新导入。
- Dock 显示简短摘要，例如“3 个成功，1 个失败”；toast 用 `warning`，不是 `destructive`。
- 全局失败时任务状态为 `failed`，toast 用 `destructive`，弹窗显示阻断原因和恢复建议。

建议错误码：

- `IMPORT_SOURCE_UNAVAILABLE`：源文件消失、不可读或浏览器文件句柄失效。
- `IMPORT_TARGET_NOT_CONFIGURED`：没有默认导入路径。
- `IMPORT_TARGET_UNAVAILABLE`：目标库路径不存在或不可写。
- `IMPORT_NOT_ENOUGH_SPACE`：目标磁盘空间不足。
- `IMPORT_CONFLICT`：目标同名文件冲突且未选择处理策略。
- `IMPORT_COPY_FAILED`：复制过程中发生普通 IO 错误。
- `IMPORT_CANCELLED`：用户取消任务。
- `IMPORT_SCAN_FAILED`：复制成功但后续扫描失败。

## 产品决策

MVP 建议默认“复制”而不是“移动”。移动源文件属于高风险操作，尤其是拖入下载目录、外置硬盘或网络盘时，误操作成本较高。后续可以在高级选项中加入“复制后删除源文件/移动源文件”，并加二次确认。

目录结构建议沿用当前整理入库语义：默认按番号建目录，保留原始扩展名。对分段文件、重复番号、目标同名文件，要进入“待确认”状态，不应静默覆盖。

导入弹窗不直接承担复杂元数据编辑。它只负责“选源文件 → 选目标库 → 复制/扫描”。影片标题、标签、演员等仍由扫描和详情页处理。

## 实施影响

前端预计新增：

- 顶栏导入入口组件，挂在 `src/layouts/AppShell.vue` 的主题切换旁边。
- `MovieImportDialog` 业务组件，放在 `src/components/jav-library`。
- 设置页存储目录卡片增加“默认导入路径”选择。
- i18n 文案：`shell.addMovie`、`import.*`、`settings.defaultImportPath` 等。

服务层预计新增：

- `LibraryService` 增加导入相关方法。
- Web adapter 调用后端导入 API。
- Mock adapter 只做本地模拟，不写磁盘。

后端预计新增：

- 设置字段：`defaultImportLibraryPathId` 或等价字段，用于记录默认导入目标。
- 导入任务 API：接收导入文件/目录信息并创建后台任务。
- 任务阶段：校验目标路径、复制文件、处理冲突、触发扫描、按需触发元数据刮削。
- 任务进度字段：至少包含 `stage`、`progress`、`currentFileName`、`copiedBytes`、`totalBytes`、`completedFiles`、`totalFiles`、`message`、`errorItems`，供弹窗和任务 Dock 复用。

## 技术风险

浏览器安全模型不允许普通网页直接把本地文件“按绝对路径”交给后端复制。实现时需要在以下方案中选择：

- 通过浏览器上传文件流到本地后端，由后端写入目标目录。实现最通用，但大视频会走 HTTP 上传链路。
- 本机路径直拷方案已按后续产品决策撤回；当前导入只保留浏览器上传链路。
- 使用 File System Access API 辅助选择文件夹。浏览器兼容性有限，仍需和本地后端写入模型对齐。

原稿表达的是目标体验；实施方案需要在确认 UI 后进一步细化技术路径。

## 待确认点

- MVP 是否只支持复制，不支持移动源文件。
- 默认导入路径是否必须从已有库路径中选择，还是允许单独配置一个“导入暂存路径”。
- 目标同名文件的默认策略：跳过、重命名、覆盖，还是导入前强制用户选择。
- 文件夹导入是否递归扫描子目录。
- 是否在导入完成后自动进入详情页/最近导入，还是只用任务 Dock 和 toast 提示。
