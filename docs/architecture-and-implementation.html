1. 总览

Curated 是本地优先的影片资料库：Vue 3 + TypeScript + Vite 作为 Web 前端，Go 守护进程（curated-backend，入口 cmd/curated）提供 HTTP API、SQLite 持久化、目录扫描、Metatube 刮削与资源缓存。长期可演进为 Electron 等桌面壳，但当前以浏览器 + 本地后端为主。

浏览器 → Vite 开发服务器（可选代理 /api） → Go Curated 后端 :8080 → SQLite + 文件系统

前端通过环境变量 VITE_USE_WEB_API 在两种数据源间切换：

  • true：调用 /api/...，经 Vite 代理到后端。
  • 非 true：使用内存 Mock 服务，无需后端。

2. 前端（Vue 3 + Vite）

2.1 入口与路由

  • src/main.ts：创建应用、挂载 router、引入全局样式（含 Tailwind v4 / shadcn 主题）。
  • src/App.vue：根组件，通常承载 router-view。
  • src/router/index.ts：使用 Hash 路由（createWebHashHistory），子路由挂在 layouts/AppShell.vue 下：
    • /library、/favorites、/recent、/tags → 同一 views/LibraryView.vue（通过路由名区分模式）。
    • /history → views/HistoryView.vue（观看历史：按本地日期分组；数据见 2.5）。
    • /detail/:id → views/DetailView.vue
    • /player/:id? → views/PlayerView.vue
    • /settings → views/SettingsView.vue

2.3 Web 适配器

src/services/adapters/web/web-library-service.ts 将 UI 需要的操作映射为 src/api/endpoints.ts 中的方法（如 listMovies、getMovie、patchMovie、startScan、getTaskStatus 等），并在适当时合并列表缓存、处理播放地址（见 src/api/playback-url.ts）。

src/api/http-client.ts 中 BASE_URL 默认为 /api，也可通过 VITE_API_BASE_URL 覆盖。

2.4 Mock 适配器

src/services/adapters/mock/mock-library-service.ts 使用内存种子数据模拟列表、详情、扫描任务等。收藏与用户评分等偏好可经 src/lib/mock-movie-prefs-storage.ts 写入 localStorage，刷新后仍保留（与真实后端 SQLite 持久化行为对应，便于离线演示）。

2.5 观看进度与历史（仅浏览器，非 API）

  • src/lib/playback-progress-storage.ts：每条影片的播放位置、时长、更新时间，键名 jav-library-playback-progress-v1。
  • src/lib/player-route.ts 与 src/lib/navigation-intent.ts：统一构造详情/播放器路由与返回语义；browse= 保存资料库上下文，back= 标记播放器返回目标，?t= 表示续播时间。
  • src/lib/playback-history-groups.ts：历史页按本地日历日分组标题（今天 / 昨天 / …）。
  • src/components/jav-library/PlaybackHistoryCard.vue：历史列表卡片（文案 + 海报 + 底栏进度条）。
  • 不与 SQLite 同步；服务端 play_history 仍为产品文档中的待决策项。

3. 后端（Go + SQLite）

3.1 进程入口

backend/cmd/curated/main.go 流程概要：

  1. 解析 -config、-mode（http / stdio / both）。
  2. config.Load 加载主配置，再 MergeLibrarySettingsFile 合并 config/library-config.cfg（organizeLibrary、metadataMovieProvider、metadataMovieStrategy、autoLibraryWatch 等）。
  3. 打开 SQLite：storage.NewSQLiteStore(cfg.DatabasePath)，执行 Migrate，必要时 SeedLibraryPathsIfEmpty。
  4. app.New 组装扫描器、刮削、任务管理器、资源缓存等；启动自动扫描循环 StartAutoScanLoop；若主配置允许目录监听且 autoLibraryWatch 为真，则 EnsureLibraryWatchRunning 启动 internal/librarywatch（fsnotify + 防抖后入队扫描）。
  5. http 模式下 server.ListenAndServe 挂载 backendApp.HTTPHandler()。

3.2 应用对象 App

backend/internal/app/app.go 中的 App 持有：

  • store *storage.SQLiteStore：影片、路径、任务、用户偏好等持久化。
  • library *library.Service：内存种子服务（开发/演示用数据集）。
  • scanner、scraper（Metatube）、assets（缓存下载）、tasks.Manager 等。

HTTPHandler() 构造 server.Handler，注入上述依赖。

3.3 HTTP 路由层

backend/internal/server/server.go 中 Routes() 注册 REST 风格接口（前缀均为 /api），主要包括：

  • GET /api/health — 健康检查（含 databasePath 等）。
  • GET /api/dev/performance — 开发环境性能摘要（整机 CPU / 后端进程 CPU），供底部性能监测条轮询。
  • GET /api/library/movies — 分页列表（query：mode、q、limit、offset）。
  • GET /api/library/movies/{movieId} — 详情。
  • PATCH /api/library/movies/{movieId} — 更新收藏与用户评分（JSON 字段见下文）。
  • GET .../stream — 视频文件流（http.ServeContent，支持 Range）。
  • POST .../scrape — 触发单部影片元数据重刮。
  • DELETE .../{movieId} — 删除影片记录及相关缓存策略由存储层实现。
  • GET/PATCH /api/settings — 设置 DTO（含 organizeLibrary、autoLibraryWatch、metadataMovieStrategy、元数据源等）。
  • GET /api/library/actors/{name}/asset/avatar — 演员头像同源缓存资源；演员资料刮削成功后会异步下载头像到本地缓存。
  • GET /api/curated-frames — 萃取帧分页查询；支持 q、actor、movieId、tag、limit、offset，响应带 total/limit/offset。
  • GET /api/curated-frames/stats、GET /api/curated-frames/tags、GET /api/curated-frames/actors — 提供萃取帧总数与标签/演员 facet 聚合，前端不再依赖全量列表自行计数。
  • POST /api/curated-frames — 兼容旧 JSON imageBase64 与新 multipart/form-data（metadata + image）上传；入库时生成缩略图 BLOB，不再因同片 3 秒阈值内的近重复帧阻断保存，重复治理改到萃取帧库页面内提示用户复核并删除多余项。
  • GET /api/curated-frames/{id}/image、GET /api/curated-frames/{id}/thumbnail — 分别提供原图与缩略图；历史行缺少缩略图时可回退原图。
  • POST /api/curated-frames/export — WebP EXIF 与 PNG iTXt 导出元数据统一补齐 tags、schemaVersion、exportedAt、appName、appVersion。
  • POST/PATCH/DELETE /api/library/paths — 库路径 CRUD。
  • POST /api/scans、GET /api/tasks/recent、GET /api/tasks/{taskId} — 扫描与任务状态。

中间层对响应做 CORS 包装，便于浏览器开发联调。

3.4 SQLite 与内存回退（重要）

列表与详情在 HTTP 层采用「数据库优先 + 空则回退内存库」策略（便于 DB 尚无数据时 UI 仍有演示数据）：

  • 列表：先 store.ListMovies；若 result.Total == 0，再调用 library.ListMovies。
  • 详情：先 store.GetMovieDetail；若 sql.ErrNoRows，再 library.GetMovie。
  • PATCH：先 store.PatchMovieUserPrefs；若返回「影片不存在于 DB 的可 PATCH 集合」且内存库存在该片，则 library.PatchMovie。

注意：GET .../stream 仅走 SQLite（打开磁盘上的视频文件），不走内存种子库；真实播放依赖扫描入库后的文件路径。

3.5 扫描、刮削与任务

扫描由 App 协调：scanner 遍历配置的库路径，解析番号、写库；若开启整理（organizeLibrary），会按规则移动/组织文件。刮削通过 Metatube SDK 拉取元数据，assets 可把封面、预览图、演员头像等缓存到 cacheDir；预览图本地缺失时，后端还会基于保存的 source_url / referer_url 做同源兜底转发。长时间操作通过 tasks.Manager 生成 taskId，前端轮询 GET /api/tasks/{taskId} 展示进度（如 ScanProgressDock.vue、use-scan-task-tracker.ts），同时任务和 provider 健康结果会带 errorCategory 等机器可读诊断字段。另：fsnotify 路径下扫描任务元数据常带 trigger: fsnotify，由 autoLibraryWatch 与主配置 libraryWatchEnabled 共同决定是否启用监听。

4. HTTP API 与契约

前后端共享的 DTO 与错误码在 Go 侧 backend/internal/contracts/ 定义；前端 TypeScript 类型在 src/api/types.ts 等与之一致（字段命名一般为 camelCase JSON）。

4.1 PATCH 影片（收藏与评分）

请求体解析逻辑在 parsePatchMovieInput：

  • isFavorite：可选布尔。
  • rating：表示用户评分；支持数字 0–5，或 null 表示清除用户评分。

数据库存储用户评分字段（如 user_rating），与刮削得到的元数据评分（如 rating）区分；列表/详情响应中会合成「有效展示分」等逻辑（见存储层与 library.Service 中的 syncEffectiveRating 一类处理）。

5. 关键数据流

5.1 打开资料库页

LibraryView → useLibraryService().listMovies → Web: api.listMovies / Mock: 内存 → 虚拟列表/瀑布流组件渲染

5.2 打开详情

DetailView → getMovie / loadMovieDetail → DetailPanel（封面、标签、MovieRatingStars、收藏等）

5.3 修改收藏或评分

UI 事件 → patchMovie / toggleFavorite → PATCH /api/library/movies/{id} → SQLite 更新 → 返回完整 MovieDetailDTO → 前端合并列表缓存

5.4 播放与续播

前端根据影片 ID 构造指向 /api/library/movies/{id}/stream 的 URL（开发环境仍走 Vite 代理）。播放器组件发起媒体请求，由 Go ServeContent 按文件实际路径读出并支持 HTTP Range。

续播（前端）：PlayerPage.vue 在 loadedmetadata 后按路由 ?t= 或 localStorage 写入的位置 seek；播放中节流回写进度。详情/资料库进播放器时，会同时携带 browse= 资料库上下文与显式 back=detail / back=browse；历史页与萃取帧页则分别使用 back=history、back=curated-frames。顶栏返回由 AppShell.vue 通过 src/lib/navigation-intent.ts 统一解析，而不是在壳层内分散拼装。

本机播放器拉起（前端）：播放器页不再默认依赖后端 native-play 去执行外部程序，而是由浏览器侧按协议模板拉起本机播放器。当前实现默认以 PotPlayer 为主，协议模板建议值为 potplayer:{url}，并保存在浏览器 localStorage（键名 curated-native-player-browser-template-v1）中，便于按机器区分是否安装了对应播放器。

7. 本地开发与联调

  • 后端：cd backend && go run ./cmd/curated，默认 :8080；数据库路径见配置与 GET /api/health。
  • 前端：pnpm dev；vite.config.ts 将 /api 代理到 http://localhost:8080。
  • 联调：根目录 .env 中 VITE_USE_WEB_API=true（或等价环境变量）。
  • 开发态诊断：src/layouts/AppShell.vue 会挂载一个固定在底部表面的 DevPerformanceBar，不影响页面布局；它聚合前端 FPS / long task / memory / 路由切换 / 解码体验指标、HTTP 请求窗口统计，以及 GET /api/dev/performance 的 CPU 摘要。
