# 后端代码梳理 (2026-03-26)

## 1. 项目概述

Curated 后端是一个 Go 编写的 HTTP/stdio 双模式服务，负责媒体库的扫描、元数据刮削、任务管理和数据持久化。

- **模块路径**: `curated-backend`
- **Go 版本**: 1.25.4
- **主要依赖**:
  - `metatube-sdk-go`: 元数据刮削
  - `modernc.org/sqlite`: SQLite 数据库
  - `go.uber.org/zap`: 结构化日志

### 3.2 关键结构体

#### App (internal/app/app.go)

核心应用结构体，协调所有组件：

```go
type App struct {
    cfg     config.Config        // 配置
    logger  *zap.Logger          // 日志
    store   *storage.SQLiteStore // 数据库
    library *library.Service     // 库内存状态
    scanner *scanner.Service     // 扫描器
    scraper scraper.Service      // 刮削器
    assets  *assets.Service      // 资源下载
    tasks   *tasks.Manager       // 任务管理

    // 功能开关(持久化到 library-config.cfg)
    organizeLibrary       bool   // 自动整理文件
    extendedLibraryImport bool   // 首次扫描布局检测
    autoLibraryWatch      bool   // 目录监听自动扫描

    scanning atomic.Bool         // 扫描互斥标志
    scrapeSem chan struct{}     // 刮削并发限制
}
```

## 4. 核心流程

### 4.1 启动流程 (cmd/curated/main.go)

1. 加载配置文件 (`config.Load`)
2. 合并库设置文件 (`config.MergeLibrarySettingsFile`)
3. 初始化日志 (`logging.New`)
4. 打开 SQLite 数据库 (`storage.NewSQLiteStore`)
5. 执行数据库迁移 (`store.Migrate`)
6. 初始化库路径种子 (`store.SeedLibraryPathsIfEmpty`)
7. 创建 App 实例 (`app.New`)
8. 启动自动扫描循环 (`StartAutoScanLoop`)
9. 启动目录监听 (`EnsureLibraryWatchRunning`)
10. 启动 HTTP/stdio 服务

### 4.2 扫描流程 (app/runScan)

```
StartScan/startLibraryScan
    ↓
创建 scan.library 任务 (pending → running)
    ↓
scanner.Scan 遍历目录
    ↓
对于每个视频文件:
    - 提取番号 (ExtractNumber)
    - 如果 organizeLibrary: 移动到 {番号}/{番号}.ext
    - 检测布局 (curated/external/loose)
    - 入库 (PersistScanMovie)
    - 如果新增/更新: 触发刮削 (enqueueScrape)
    ↓
任务完成 (completed/failed)
```

### 4.3 刮削流程 (app/runScrape)

```
enqueueScrape (受 scrapeSem 并发限制)
    ↓
创建 scrape.movie 任务
    ↓
scraper.Scrape (Metatube 搜索+详情)
    ↓
保存元数据 (SaveMovieMetadata)
    ↓
如果 organizeLibrary: 写入 NFO
    ↓
触发资源下载 (runAssetDownload)
    ↓
下载海报/缩略图到 cacheDir 或影片目录
```

### 4.4 任务生命周期

任务状态流转:

```
pending → running → completed
                    ↘ partial_failed
                    ↘ failed
                    ↘ cancelled
```

任务类型:
- `scan.library`: 库扫描
- `scrape.movie`: 单影片元数据刮削
- `scrape.actor`: 演员资料刮削
- `asset.download`: 资源下载

## 8. 特色功能实现

### 8.1 番号解析 (scanner/number.go)

支持格式:
- 标准格式: `ABC-123`, `ABC_123` → `ABC-123`
- 小写格式: `abc123` → `ABC-123`
- 特殊前缀: `FC2`, `heyzo`, `tokyo-hot`, `1pondo`, `caribbeancom`

### 8.2 文件整理 (library/organize.go)

当 `organizeLibrary=true` 时:
```
原路径: /library/ABC-123.mp4
整理后: /library/ABC-123/ABC-123.mp4
        /library/ABC-123/movie.nfo
        /library/ABC-123/poster.jpg
```

### 8.3 目录监听 (librarywatch/)

- 使用 `fsnotify` 监听库根目录
- 防抖机制(默认 1500ms)合并连续事件
- 仅当 `autoLibraryWatch=true` 且主配置允许时启用
- 触发扫描时标记 `trigger: fsnotify`

### 8.4 回收站 (movie_trash_repository.go)

- 软删除: 设置 `trashed_at` 字段
- 回收站视图: `mode=trash` 查询
- 恢复: 清空 `trashed_at`
- 彻底删除: `DELETE ?permanent=true` (需先移入回收站)
